# 🚁 Helicopter Delivery System

## Description

Консольное приложение на C++ для учёта грузовых вертолётов, экипажей и рейсов в горном районе (лабораторная работа № 4, вариант 3). Реализованы вход в систему с разграничением ролей (командир авиаотряда и член экипажа), выборки по заданию (налёт после капремонта, рейсы за период, сводки по спец/обычным рейсам, аналитика по вертолётам и экипажам), начисление вознаграждений за период с записью в служебную таблицу и расчёт выплат конкретному летчику, в том числе отдельно по типу рейса. Поддерживаются CRUD для основных сущностей (для командира), экспорт данных в CSV и JSON, хранение изображений в SQLite (BLOB). База данных — SQLite; контроль превышения летного ресурса после последнего капремонта выполняется триггером при добавлении рейса.

## Installation

### 🔧 Требования
- Компилятор C/C++ (gcc / clang / MSVC)
- SQLite
- Git

### Установка
```bash
git clone https://github.com/fpmi-tpmp2026/tpmp-gr11a-lab4-gpt-dreamteam
cd tpmp-gr11a-lab4-gpt-dreamteam
mkdir -p build
   g++ -std=c++17 -Wall -Wextra -O2 -I includes src/*.cpp -lsqlite3 -o build/helicopter_app
```
**Первый запуск с демо-данными** (создаёт `build/helicopter.db` и подставляет тестовых пользователей и справочники):

   ```bash
   export HELI_SEED=1
   ./build/helicopter_app
   ```
   Если скрипты SQL лежат не в каталоге `sql` относительно текущей директории, укажите путь:

   ```bash
   export HELI_SQL_DIR=/абсолютный/путь/к/sql
   ```
При необходимости передайте путь к файлу БД аргументом: `./build/helicopter_app /tmp/my.db`.
## Usage

- Запустите `./build/helicopter_app` (или с путём к БД). Введите логин и пароль.
- **Демо-аккаунты при `HELI_SEED=1`:** `commander` / `cmd_secret` (командир), `pilot_ivanov` / `pilot1` (экипаж, таб. № 1001).
- **Командир** видит пункты аналитики по всему отряду, начисления за период в таблицу `AIR_PAYROLL_PERIOD`, CRUD вертолётов и экипажа, управление пользователями, экспорт CSV/JSON.
- **Член экипажа** может запрашивать пункты, помеченные в меню как доступные экипажу (`*`), и расчёт своих начислений за период (табельный номер подставляется из сессии).
- Даты вводите в формате **YYYY-MM-DD**. Стоимость рейсов и выплат в приложении задаётся в условных единицах (копейки), проценты из `AIR_CONFIG`: 5% за обычный рейс, 10% за спецрейс (согласно варианту).
- Справка по аргументам: `./build/helicopter_app --help`.

Подробная постановка, глоссарий, требования и схема БД — в [вики репозитория документации](wiki/Home.md) (или в разделе Wiki на GitHub после загрузки страниц).
